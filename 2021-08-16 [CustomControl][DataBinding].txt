 ㅁ custom control 생성. 
    -사용자정의 (user contorl) 사용자 지정(costom control )의 차이 
       1) 상속구조 다름 -> 상속을 덜 받은 custom control 의 자유도가 높다.
	UserControl : UserControl<-ContentControl<-Control(IaddChild *인터페이스)  
       	CustomControl : Control
       2) 1)로 인해 몇몇 객체 내부 이벤트(ex addchild) 및 프로퍼티를 직접 변경할 수 있다. 
       3) 상속을 덜 받아 가볍다. 
    - 손이 덜 가고 편하고 단순한 컨트롤은 UserControl을 사용하면 되지만 특화된 기능으은 CustomControl을 사용한다. 
    - Custom Control을 수정하면 해당 프로젝트를 빌드해야 main view에서 보인다.

 ㅁ Custom Control 등록 
    -  Custom Control을 사용할 마크업 파일의 루트에 xmlns:MyNamespace="clr-namespace:CCBtn;assembly=CCBtn" 추가. 
    -  <MyNamespace:CustomControl1/> 를 추가해서 사용하면된다. 

 ㅁ Data Binding
   - 데이터 바인딩은 소스와 타겟으로 구성.  일반적으로 소스는 데이터(ViewModel)이고 타겟은 컨트롤 이다. 
   - 타겟 객체는 UI 컨트롤을 소유하는 객체이고, 소스객체는 ViewModel이다.
      -> cs파일에서 DataContext를 this로 하면 ui에서 viewmodel의 객체를 소스로 쓸수 있게 바인딩된다.
